  <action>
Add a loading overlay on top of the log table when filters are being applied. Currently, SkeletonRows appear below the table during loading, but there's no visual feedback that the existing table data is being refreshed.

**Changes to log-list.tsx (around line 62-73):**

Wrap the LogTable component in a relative positioned container with conditional overlay:

```tsx
{/* Log Table with Loading Overlay */}
<div className="relative">
  <LogTable
    logs={logs}
    onLoadMore={loadMore}
    hasMore={hasMore}
    isLoading={isLoading}
    sort={filters.sort || 'timestamp'}
    order={filters.order || 'desc'}
  />

  {/* Loading overlay - shown during filter updates */}
  {isLoading && logs.length > 0 && (
    <div className="absolute inset-0 bg-white/60 backdrop-blur-sm flex items-center justify-center rounded">
      <div className="bg-white px-6 py-3 rounded-lg shadow-lg border flex items-center gap-3">
        <div className="animate-spin h-5 w-5 border-2 border-slate-300 border-t-slate-600 rounded-full" />
        <span className="text-sm font-medium text-slate-700">Updating filters...</span>
      </div>
    </div>
  )}
</div>
```

**Remove the standalone SkeletonRows** (line ~72):
Delete or comment out the line `{isLoading && <SkeletonRows count={3} />}` since the overlay provides better feedback.

**Why this approach:**
- `isLoading && logs.length > 0` condition ensures overlay only shows during filter updates (not initial load)
- Overlay with backdrop-blur provides clear visual feedback without hiding content completely
- Spinner + text message ("Updating filters...") is more informative than skeleton rows
- absolute positioning keeps the overlay above the table without layout shift
- Centered loading indicator draws user attention to the loading state
- Semi-transparent white overlay (bg-white/60) maintains context while signaling refresh

**Visual behavior:**
- Initial page load: Table appears with data (from SSR), no overlay
- Filter change: Overlay appears over existing table, shows spinner + message
- Filter load complete: Overlay fades out, new filtered results visible
  </action>
