## Interaction Patterns

### Log List Table
- **Click row:** Open detail modal
- **Hover row:** Change background to slate-100
- **Click column header:** Toggle sort (ascending ↔ descending)
- **Scroll to bottom:** Auto-load next page (infinite scroll)

### Filter Sidebar
- **Change search input:** Debounced 300ms, then fetch filtered results
- **Select severity:** Immediate fetch with updated filters
- **Select source:** Immediate fetch with updated filters
- **Change date range:** Immediate fetch when second date selected
- **Click filter chip X:** Remove that filter, fetch updated results
- **Click Reset Filters:** Clear all filters, fetch unfiltered results

### Log Detail Modal
- **Click outside modal:** Close modal
- **Press Escape:** Close modal
- **Click Close button:** Close modal
- **Modal open:** Body scroll locked, modal centered with overlay

### Create Form
- **Submit form:** Optimistic UI (add to table immediately), show skeleton then real data
- **Validation error:** Show toast with error message
- **Success:** Show success toast, redirect to /logs with new log visible

### URL State
- **Filter changes:** Update URL search params (?search=error&severity=ERROR)
- **Sort changes:** Update URL search params (?sort=timestamp&order=desc)
- **Browser back/forward:** Restore filter/sort state from URL

## Loading States

### Initial Page Load
- Sidebar: Rendered immediately (no loading state)
- Table: 10 skeleton rows (gray pulse animation, maintains table structure)
- Filters: Rendered immediately with default values

### Infinite Scroll Load More
- Append 5 skeleton rows at bottom of table
- Load data, replace skeleton with real rows
- No spinner or interruption to existing content

### Filter/Sort Change
- Keep existing rows visible with 50% opacity overlay
- Show 3 skeleton rows at top
- Replace all content once data loads (prevents layout shift)

### Form Submission
- Button shows loading spinner inside button
- Form fields disabled during submission
- Optimistic update: Add skeleton row to table immediately

---

## Error States

### Network Error
- Toast notification (top-right): "Unable to connect. Retrying..."
- Auto-retry after 2 seconds (max 2 attempts)
- If all retries fail: Toast changes to "Failed to load. Please refresh."
- Table shows last successfully loaded data (if any)

### Validation Error
- Toast notification (top-right): "Please check your input and try again."
- Highlight invalid form fields with red border
- Show validation message below field

### Empty Search Results
- Clear existing skeleton rows
- Show centered message: "No logs match your filters"
- Show "Reset Filters" button below message

### API Error (4xx/5xx)
- Toast notification with error message from API response
- Include request_id if available in response
- Preserve user's filter/sort state

## Accessibility

### Keyboard Navigation
- Tab through all interactive elements (filters, table rows, buttons)
- Enter/Space on table row: Open detail modal
- Escape in modal: Close modal
- Arrow keys in dropdowns: Navigate options

### Screen Reader
- Table has proper <thead> and <th> elements with scope
- Skeleton rows have aria-label="Loading"
- Modal has aria-labelledby pointing to modal title
- Toast notifications use aria-live="polite"
- Filter form has proper <label> for each input
- Active filter chips announce "Remove [filter name] filter"

### Focus Management
- Modal opening: Focus moves to close button
- Modal closing: Focus returns to trigger element (table row)
- Form submission: Focus moves to first error field (if error)
- Page load: Focus starts at search input (convenience)

### Color Contrast
- All text meets WCAG AA standards (4.5:1 for body, 3:1 for large text)
- Severity badges: Sufficient contrast between background and text
- Hover states: Visible but subtle (no contrast issues)

## Performance Targets

### Initial Page Load
- First Contentful Paint: < 1.5s
- Time to Interactive: < 3s
- Largest Contentful Paint: < 2.5s

### Interaction Performance
- Filter change to visible results: < 500ms
- Sort change to re-rendered table: < 300ms
- Table row hover: 60fps (no jank)
- Infinite scroll trigger to new data: < 500ms

### Data Limits
- Initial fetch: 50 logs (API default limit)
- Infinite scroll batch: 50 logs per fetch
- Virtual scrolling: Render only visible rows + buffer (performance with 100k logs)

### Bundle Size
- Initial JavaScript: < 200KB gzipped
- Route-based code splitting: Lazy-load modal and create form
- Tree-shaking: shadcn components are modular (only import what's used)

## Technical Constraints

### Browser Support
- Chrome 90+ (primary target)
- Firefox 88+
- Safari 14+
- Edge 90+
- No IE11 support

### API Integration
- Base URL: `http://localhost:8000/api` (configurable via `NEXT_PUBLIC_API_URL`)
- Cursor pagination: Opaque tokens (no page numbers)
- Error format: `{"detail": "message", "request_id": "uuid"}`
- Timestamp format: ISO 8601 with timezone (e.g., "2024-03-20T15:30:00Z")

### State Management
- URL as source of truth for filters/sort
- React Context for modal state (open/closed + log data)
- No Redux or Zustand (Next.js native state sufficient)

### Data Fetching
- Server Components for initial page data (SSR)
- Client Components for interactive filters (CSR)
- No SWR or React Query (native fetch with Next.js cache)
