```typescript
const TIME_RANGES = [
  { label: 'Last hour', value: '1h', fn: () => subHours(new Date(), 1) },
  { label: 'Last 6 hours', value: '6h', fn: () => subHours(new Date(), 6) },
  { label: 'Last 24 hours', value: '24h', fn: () => subHours(new Date(), 24) },
  { label: 'Last 7 days', value: '7d', fn: () => subDays(new Date(), 7) },
  { label: 'Last 30 days', value: '30d', fn: () => subDays(new Date(), 30) },
  { label: 'Custom', value: 'custom', fn: null }
]
```

3. **State management:**
   - Use `useQueryStates` to manage `date_from` and `date_to` URL parameters (parseAsString)
   - Local state `selectedRange` to track which preset is active ('1h' | '6h' | '24h' | '7d' | '30d' | 'custom')
   - Local state `customFrom` and `customTo` for custom date picker values (format: yyyy-MM-dd'T'HH:mm for datetime-local input)

4. **Segmented button group:**
   - Horizontal flex container with connected buttons (border-collapse pattern)
   - Each button:
     - `variant="outline"` when not selected
     - `variant="default"` (filled background) when selected
     - onClick handler that:
       - For presets (not 'custom'): Calculate dates using preset.fn(), convert to ISO string, update URL state via setQuery
       - For 'custom': Set selectedRange to 'custom' to show date pickers, don't update URL yet
   - Styling: First button rounded-l-md, last button rounded-r-md, middle buttons square, shared borders (-ml-px pattern)

5. **Custom date picker section:**
   - Conditionally render when `selectedRange === 'custom'`
   - Card wrapper with p-4 padding, appears below button group with mt-4 spacing
   - Two datetime-local inputs (From / To) with Label components
   - "Apply" button that validates customFrom < customTo, then updates URL state
   - Format datetime-local values as `yyyy-MM-dd'T'HH:mm` using date-fns format()
   - Initialize customFrom/customTo from URL params if they exist

6. **Effect to sync URL to selected range:**
   - useEffect that detects which preset matches current URL date_from/date_to
   - If no match, set selectedRange to 'custom'
   - This ensures refreshing the page maintains correct button selection

**CRITICAL - URL state pattern:**
```typescript
const [{ date_from, date_to }, setQuery] = useQueryStates({
  date_from: parseAsString,
  date_to: parseAsString
})

// Update URL on preset click
const handlePresetClick = (preset) => {
  if (preset.value === 'custom') {
    setSelectedRange('custom')
    return
  }

  const from = preset.fn()
  const to = new Date()

  setQuery({
    date_from: from.toISOString(),
    date_to: to.toISOString()
  })
  setSelectedRange(preset.value)
}
```
