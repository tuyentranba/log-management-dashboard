## Decisions Made

1. **Reused existing CreateForm with optional callback props** - Maintained single source of truth for create logic. Optional onSuccess/onCancel props enable context-aware behavior: modal context calls callbacks, page context uses router.push. Cleaner than explicit isModal boolean flag.

2. **Conditional submission overlay** - Added `showOverlay = !onSuccess && isSubmitting` to only show overlay in page context. Modal already has Dialog backdrop, so overlay would create double-blur effect. Button spinner sufficient in modal.

3. **Separate Client Components for interactivity** - Created CreateButton and CreateLogModal as Client Components to keep page.tsx as async Server Component. Preserves SSR benefits for initial data fetch while enabling interactive elements via Client Component islands pattern.

4. **Modal closes without warning** - No unsaved changes dialog per Phase 05.3 CONTEXT.md decisions. Users can quickly open/close modal to retry. Matches edit/delete modal behavior from Phase 05.2.

5. **Removed /create route completely** - Single path to create logs (modal from /logs header) simplifies mental model. No sidebar link, no standalone page. All CRUD operations now accessible from /logs via modals.
