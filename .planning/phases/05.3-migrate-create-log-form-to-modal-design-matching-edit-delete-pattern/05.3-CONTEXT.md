# Phase 05.3: Migrate Create Log Form to Modal - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Move the create log form from standalone `/create` page to a modal accessible from `/logs` page. Modal follows the same pattern as edit/delete modals established in Phase 05.2 (Dialog component, URL state management, toast notifications, auto-refresh). The `/create` route and sidebar link will be removed entirely.

</domain>

<decisions>
## Implementation Decisions

### 1. Modal Trigger Placement

**Decision:** Header button next to Export CSV

**Details:**
- Add "+ Create Log" button in logs page header
- Position: Primary action (Create Log) comes before Export CSV button
- Button label: "+ Create Log" with plus icon (e.g., lucide-react Plus icon)
- Visual style: Primary button (default variant)
- Layout: Left side of header (title), right side (Create Log | Export CSV)

**Rationale:**
- Keeps all actions together in one place
- Consistent with existing header layout
- Primary action positioned first (standard UI convention)

### 2. Form Component Reuse

**Decision:** Reuse existing CreateForm with optional callback props

**Details:**
- Adapt `frontend/src/app/create/_components/create-form.tsx` for both contexts
- Add optional props: `onSuccess?: () => void` and `onCancel?: () => void`
- If callbacks provided (modal context): call them instead of router.push
- If callbacks not provided (page context): use existing router.push behavior
- Remove submission overlay when in modal context (button spinner sufficient)
- Keep overlay for page context (current behavior)

**Implementation pattern:**
```typescript
interface CreateFormProps {
  onSuccess?: () => void
  onCancel?: () => void
}

export function CreateForm({ onSuccess, onCancel }: CreateFormProps = {}) {
  const router = useRouter()

  const handleSuccess = () => {
    toast.success('Log created successfully')
    if (onSuccess) {
      onSuccess() // Modal: close and refresh
    } else {
      router.push('/logs') // Page: navigate
    }
  }

  // Show overlay only when NOT in modal context
  const showOverlay = !onSuccess && isSubmitting
}
```

**Rationale:**
- DRY principle — single source of truth for create logic
- Optional props pattern cleaner than explicit isModal flag
- Maintains backward compatibility with page usage
- Simpler than extracting shared field components

### 3. Navigation Changes

**Decision:** Remove /create route and sidebar link completely

**Details:**
- Delete `frontend/src/app/create/` directory and all contents
- Remove "Create" link from sidebar navigation component
- Users access create functionality only via "+ Create Log" button in logs header
- URL state: `?create=true` triggers modal open (matches `?log=123` pattern)

**Rationale:**
- Single path to create logs (modal) — simpler mental model
- No broken links concern (demo application)
- Reduces maintenance burden (one less route to maintain)
- Sidebar remains clean with just Logs and Analytics links

### 4. Modal Behavior

**Decision:** Close without warning, no cancel button, auto-refresh on success

**Details:**
- **Close behavior:** Click outside modal or press Escape closes immediately (no unsaved changes warning)
- **Cancel button:** None — users close via outside click or Escape
- **Modal title:** "Create Log"
- **Success flow:**
  1. API call completes successfully
  2. Close modal (`setIsCreating(false)` clears `?create=true`)
  3. Show toast: "Log created successfully"
  4. Trigger list refresh via `onRefetch()` callback
  5. New log appears in list

**UX Flow:**
```typescript
const [isCreating, setIsCreating] = useQueryState('create', parseAsBoolean)

const handleCreateSuccess = () => {
  toast.success('Log created successfully')
  setIsCreating(false) // Close modal
  onRefetch() // Refresh log list
}

// In modal:
<Dialog open={isCreating} onOpenChange={(open) => !open && setIsCreating(false)}>
  <DialogContent className="max-w-2xl">
    <DialogHeader>
      <DialogTitle>Create Log</DialogTitle>
    </DialogHeader>
    <CreateForm onSuccess={handleCreateSuccess} />
  </DialogContent>
</Dialog>
```

**Rationale:**
- Close without warning: Standard dialog behavior, reduces friction
- No cancel button: Cleaner UI, outside click sufficient
- Auto-refresh: Matches edit/delete pattern from Phase 05.2
- Consistent UX across all CRUD operations

### Claude's Discretion

- Modal width and responsive behavior
- Error state handling during creation (toast message wording)
- Focus management when modal opens (first form field vs auto-focus behavior)
