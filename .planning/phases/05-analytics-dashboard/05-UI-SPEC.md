## Interaction Patterns

| Interaction | Behavior | Rationale |
|-------------|----------|-----------|
| Date preset click | Updates date_from and date_to in URL params, triggers data refetch | Fast common selections (05-CONTEXT.md) |
| Custom date input | Updates date range in URL params on change, triggers data refetch | Power user feature for arbitrary ranges |
| Severity filter | Updates severity param in URL, triggers data refetch | Reuses Phase 3 filter pattern |
| Source filter | Updates source param in URL, triggers data refetch | Reuses Phase 3 filter pattern |
| Time-series hover | Shows tooltip with bucket time + count | 05-CONTEXT.md: tooltip only, no click |
| Time-series click | No action (tooltip only) | 05-CONTEXT.md decision |
| Severity bar hover | Shows tooltip with severity + count + percentage | Enhanced feedback |
| Severity bar click | Navigates to /logs?severity=[clicked severity] | 05-CONTEXT.md: drill down to filtered list |
| "View All Logs" click | Navigates to /logs with current date/severity/source filters | Context preservation |
| Page load | Auto-applies "Last 7 Days" default if no date range in URL | 05-CONTEXT.md default range |

**Source:** 05-CONTEXT.md interaction decisions, Phase 3 URL state patterns (nuqs)

### Design Decisions Made by Researcher

Following "Claude's Discretion" from 05-CONTEXT.md:
- Loading skeleton uses pulsing background (consistent with Phase 3 patterns)
- Empty state message placement: centered in chart area with axes visible
- Zero-count severity bars: show all 4 bars, zero-height bars still display label
- Chart animations: Use Recharts default (300ms, can be adjusted during implementation)
- Tooltip styling: Recharts default with custom formatters for date/count/percentage
- Error state: Full-width alert above charts (not modal, allows retry without dismissing)

### Implementation Notes

- All charts must be Client Components ('use client') due to Recharts browser-only rendering
- Date range must be passed as UTC to backend, displayed in local timezone on frontend
- Severity bar chart uses Cell component to apply individual colors to each bar
- Time-series chart auto-adjusts X-axis format based on granularity (hour: "MMM d, HH:mm", day: "MMM d")
- "View All Logs" button includes current filters in navigation URL for context preservation
